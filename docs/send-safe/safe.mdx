# Send Safe

Send Safe is a multisig on Canton Network. It's built for Canton teams that want passkey-grade security with treasury-level controls — require multiple approvers on any transfer while keeping a familiar biometric UX.

## What Send Safe Provides

- **Shared ownership:** Any asset inside a Send Safe requires a quorum before it can move.
- **Device-level security:** Every signer authenticates with their own passkeys, removing the need to manage seed phrases or browser extensions.
- **Policy transparency:** Thresholds, roles, and pending transactions are always visible to every signer in the safe dashboard.
- **Private settlement:** Transactions execute on Canton Network, so counterparties and amounts stay confidential even when multiple signers participate.

## Create a Send Safe

1. Open Send Safe, navigate to **Multisig**, and select **Create Multisig**.
2. Name the multisig and pick a unique name.
3. Define the multisig's signing threshold: `m-of-n` (default to majority).
4. Add signer party ids or usernames. Invitations stay pending until each signer accepts.
5. Fund the multisig by transferring assets from an existing account or importing an onchain balance using the provided deposit address.

A Send Safe stays in a draft state until the final signer joins and the funding transaction finalizes. Draft safes cannot hold assets.

## Using a Send Safe

We designed Send Safe to feel like using your primary account. Once your safe is active, swap to your Send Safe account in the party selector in the navigation bar. From there you can propose and send transactions as you normally would in the app.

## Managing a Send Safe

- **Add signer:** Requires an existing quorum to approve and new signers must accept the invite.
- **Remove signer:** Requires existing quorum to approve.
- **Modify threshold:** Requires existing quorum to approve.
